# 01 · Foundations

아바팅 디자인의 근본 원칙. 이 문서는 개별 화면 디자인보다 **상위 규약**이다.

## 브랜드 포지셔닝

- **컨셉**: 인터랙티브 관찰 예능형 소셜 게임
- **타겟 정서**: "감정 과잉 없이 객관적으로 관찰하고 판단한다"
- **연상 키워드**: 개발자 도구, 관제 시스템, 분석 대시보드, Linear

## 디자인 철학

### 3대 원칙

1. **관조적 (Observational)**
   - 사용자는 "참여자"가 아닌 "관찰자"다.
   - 감정적 몰입을 강요하는 연출을 피하고, 데이터·사실·숫자를 전면에 둔다.
   - 호감도는 하트 애니메이션이 아닌 숫자와 프로그레스 바로 표현한다.

2. **절제된 (Restrained)**
   - "빼는 것"이 "더하는 것"보다 항상 우선한다.
   - 하나의 포인트 컬러(brand)만 사용하며, 다른 색상은 기능 신호(success/warning/danger)에만 할당한다.
   - 장식적 요소(그라데이션, 글로우, 파티클)는 전면 금지한다.

3. **구조적 (Structural)**
   - 모든 요소는 명확한 그리드와 위계를 따른다.
   - 간격 체계는 4px 그리드에서 벗어나지 않는다.
   - 레이아웃은 예측 가능해야 한다 (비대칭, 대각선 흐름 금지).

## Anti-AI Aesthetics — 금지 사항

AI가 생성한 티가 나는 전형적 패턴을 **명시적으로 금지**한다.

### 컬러 금지 사항
- 보라색 그라데이션 (`linear-gradient(purple, pink)` 류)
- 네온 글로우 (`box-shadow: 0 0 30px <bright-color>`)
- 형광/쨍한 색상 (채도 90% 이상)
- 다중 컬러 그라데이션 (3색 이상)
- 배경 전체를 덮는 그라데이션

### 레이아웃 금지 사항
- "Hero 섹션 + 3카드" 클리셰 레이아웃
- 중앙 정렬 큰 제목 + 서브타이틀 + CTA 버튼 (온보딩 첫 화면 제외)
- 떠다니는 요소 (`animation: float`)
- 패럴랙스 스크롤
- 대각선 섹션 구분

### 컴포넌트 금지 사항
- 20px를 초과하는 border-radius
- 과도한 `backdrop-filter: blur`로 "글래스모피즘"
- 컴포넌트 호버 시 `scale(1.05)` 이상 확대
- 박스 그림자 다중 레이어 (3개 이상 중첩)

### 타이포그래피 금지 사항
- Serif 폰트 사용 (이탤릭 강조 문구 포함)
- 무지개 그라데이션 텍스트 (`background-clip: text`로 여러 색상 사용)
- 과도한 letter-spacing (>1px for body, >3px for display)
- Display 폰트 사용 (Playfair, Dancing Script 류)

### 카피라이팅 금지 사항
- "✨", "🚀", "💫" 등 이모지로 강조
- "신비로운", "마법 같은", "설렘 가득한" 감정 과잉 표현
- 느낌표 남용 ("지금 시작하세요!")
- "AI가 만든 티가 나는" 전형적 카피 ("당신의 운명을 찾아드릴게요")

## Tone of Voice

### 카피라이팅 원칙

- **사실 기반**: "평균 호감도 64/100" (O) vs "엄청난 호감도!" (X)
- **간결**: 문장은 한 줄에 끝난다. 두 줄이 필요하면 두 문장으로 나눈다.
- **기능 중심**: "아바타가 대화를 시작했습니다" (O) vs "드디어 첫 만남이 시작됐어요!" (X)
- **개발자 도구 감각**: `TURN 06 / 12` · `+6 지난 턴 대비` · 모노스페이스 메타

### 예시: 같은 상황의 Before/After

| 상황 | ❌ Before (AI스러운) | ✅ After (확정) |
|------|---------------------|----------------|
| 매칭 성공 | "🎉 운명의 상대를 만났어요!" | "호감도 임계값 돌파" |
| 훈수 권유 | "💡 지금이 결정적 순간이에요!" | "현재 +6pt 상승 중 · 개입 효과 높음" |
| 온보딩 완료 | "✨ 당신만의 특별한 아바타가 탄생했어요!" | "당신의 아바타가 준비됐어요" |
| 빈 상태 | "아직 아바타가 없어요 🥺 첫 아바타를 만들어볼까요?" | "아바타 없음 · 설문 완료 후 생성됩니다" |

## 디자인 결정 시 체크리스트

새 컴포넌트/화면을 만들 때 반드시 확인:

- [ ] 이 화면에 이모지가 있는가? → 제거하고 Lucide 아이콘으로 대체
- [ ] 그라데이션이 있는가? → 단일 색상으로 변경
- [ ] border-radius가 12px를 초과하는가? → 12px 이하로 축소
- [ ] 감정적 카피가 있는가? → 사실 기반으로 재작성
- [ ] 색상이 3개 이상 사용되었는가? → brand + neutral + 기능색 하나로 축소
- [ ] "float", "pulse", "glow" 애니메이션이 있는가? → 제거 또는 기능적 목적(typing, shimmer)만 허용
- [ ] "Hero + 3카드" 구조인가? → 테이블·리스트·대시보드 레이아웃으로 재설계

이 체크리스트를 통과하지 않으면 그 디자인은 **승인되지 않는다**.
