# DSD · 내 아바타 (프로필 편집)

## 1. 개요

**목적**: 내 아바타의 스탯, 태그, 이름을 조정하고, 필요 시 설문/GPT 연결을 재실행.
**성공 지표**: 스탯 조정 후 평균 호감도 변화, 태그 편집 빈도.

## 2. User Journey

```
Dashboard / Sidebar → [My Avatar] → Edit → Save → back to previous
```

## 3. 패턴

`B (App Shell)` + 탭 네비게이션.

## 4. 레이아웃

```
┌────────┬──────────────────────────────────────┐
│Sidebar │ 홈 > 내 아바타                        │
│        ├──────────────────────────────────────┤

│        │ ┌──────────────────────────────────┐ │
│        │ │ [HW] hyunwoo                    │ │
│        │ │ @hyunwoo · Lv.4 · [내향·분석형]  │ │
│        │ │                 [편집 모드 토글]  │ │
│        │ └──────────────────────────────────┘ │
```

### 각 탭 내용

#### 탭 1 · 기본 정보
- 이름 (Input, max 12)
- 한 줄 소개 (Input, max 60)
- 표시 이니셜 (자동 도출, 수동 override 가능 — 2글자)

#### 탭 2 · 스탯
- 4개 StatBar, 편집 모드일 때 드래그 가능한 슬라이더로 변환
- 우측 "설문 다시하기" 링크 (→ onboarding-survey, 결과 덮어쓰기 경고)

#### 탭 3 · 태그
- 현재 태그 리스트 (X 버튼으로 삭제 가능)
- "태그 추가" 입력창 (자동완성, 기존 태그풀에서 제안)
- 우측 "GPT로 다시 학습" 버튼 (→ onboarding-connect, 새 Connect Code)
- 최대 8개 제한

#### 탭 4 · 고급
- 활동 상태 (온라인/쉬는 중 토글)
- 매칭 설정 (선호 호감도 범위, 대화 스타일 선호도)
- 아바타 초기화 (위험 액션 — 확인 모달)

## 5. 컴포넌트

**재사용**: Button, Input, Tag, AvatarMark, StatBar, Icon

**신규**:
- `StatSlider`: StatBar의 편집 가능 variant. 드래그·키보드 입력 지원.
- `Tabs`: 탭 네비게이션 (4개 탭). 쿼리스트링으로 상태 보존.

## 6. 디자인 토큰

- 탭 아래 구분선: `border-b --border`
- 활성 탭: `border-b-2 --brand`
- Sticky footer (변경사항 알림): `--bg-elev-2` border-top, padding `space-4`

## 7. 타이포그래피

| 요소 | 스타일 |
|------|--------|
| 프로필 헤더 이름 | `title` |
| 탭 레이블 | `ui-label` (500) |
| 활성 탭 | `ui-label` (600) + 하단 브랜드 라인 |
| 설정 필드 라벨 | `ui-label` `--text-2` |

## 8. 아이콘

| 위치 | Lucide |
|------|--------|
| 편집 모드 토글 | `Settings` (off) / `Check` (save mode) |
| 탭 비활성 → 활성 전환 | 없음 (텍스트만) |
| "설문 다시하기" | `ArrowUpRight` |
| "GPT 재학습" | `Sparkles` |
| 태그 삭제 | `X` (12px) |
| 저장 | `Check` (16) |
| 취소 | 없음 |

## 9. 인터랙션

### 편집 모드

- **진입**: 우상단 "편집" 버튼 클릭 → 모든 필드가 readable→editable 상태로 전환
- **이탈 (저장 없이)**: 변경사항 있으면 "저장하지 않은 변경사항이 있습니다" 확인 모달
- **저장**: API 호출 → 성공 시 편집 모드 종료 + 토스트 "저장되었습니다"

### Sticky Footer (변경 감지)

- 모든 필드에 `dirty` flag 추적
- 1개 이상 dirty면 하단 footer 표시 (slide-up 200ms)
- footer: 변경된 필드 수 요약 ("3개 항목 변경") + 저장/취소

### 위험 액션 (탭 4)

- **아바타 초기화**: 확인 모달 (타이핑 확인 요구: 닉네임 재입력)
- **활동 상태 토글**: 즉시 적용 (footer 거치지 않음)

## 10. 반응형

- Mobile: 탭을 가로 스크롤 가능한 chip으로 전환
- StatSlider: 터치 드래그 + 좌우 +/- 버튼
- Sticky footer: 화면 하단 고정, safe-area 고려

## 11. 접근성

- 탭 네비게이션: `role="tablist"` · 화살표 키로 탭 이동
- StatSlider: `role="slider"` `aria-valuenow` `aria-valuemin=0` `aria-valuemax=100`
- 편집 모드 토글: `aria-pressed={isEditing}`

## 12. 카피라이팅

### 탭 라벨

- 기본 정보 · 스탯 · 태그 · 고급

### 탭 1 · 기본 정보

- 이름 라벨: 이름
- 이름 helper: 2–12자 · 영문/숫자/한글
- 소개 라벨: 한 줄 소개
- 소개 helper: 최대 60자 · 상대가 프로필을 볼 때 표시됩니다
- 이니셜 라벨: 표시 이니셜 (2글자)
- 이니셜 helper: 이름에서 자동 추출됩니다. 직접 지정 가능.

### 탭 2 · 스탯

- 제목: Stats
- 서브: 매칭 알고리즘에 직접 영향을 줍니다. 신중히 조정하세요.
- "설문 다시하기" 링크: 설문으로 다시 측정하기
- 링크 설명: 기존 답변을 불러와 수정할 수 있어요

### 탭 3 · 태그

- 제목: Affinity Tags
- 서브: 관심사·성향 키워드. 매칭 시 공통 태그가 호환도에 반영됩니다.
- 태그 개수 표시: {n} / 8
- 입력 placeholder: 새 태그 추가...
- "GPT 재학습" 버튼: Custom GPT와 다시 대화
- 버튼 설명: 10분 분량의 대화로 스탯과 태그를 다시 추출합니다

### 탭 4 · 고급

- 활동 상태 레이블: 활동 상태
- 옵션: 활성 · 쉬는 중
- 활성 설명: 다른 사용자가 내 아바타를 파견/관전할 수 있습니다
- 쉬는 중 설명: 새 파견 요청을 받지 않습니다. 진행 중인 세션은 영향 없음.
- 위험 액션 제목: 아바타 초기화
- 위험 설명: 스탯, 태그, 활동 기록이 모두 삭제되며 복구할 수 없습니다. 매칭된 본캐 연결은 유지됩니다.

### Sticky footer

- "{n}개 항목 변경됨"
- 저장 / 취소

### 토스트/모달

- 저장 성공: "저장되었습니다"
- 저장 실패: "저장에 실패했습니다. 다시 시도해주세요."
- 편집 중 이탈: "저장하지 않은 변경사항이 있습니다. 계속하시겠어요?" / "나가기" / "머무르기"
- 초기화 확인 모달:
  - 제목: 아바타 초기화
  - 내용: 이 작업은 되돌릴 수 없습니다. 아바타의 모든 스탯, 태그, 활동 기록이 삭제됩니다. 계속하려면 닉네임 '{nickname}'을 입력하세요.
  - 확인 버튼 (비활성 → 정확히 입력 시 활성): 초기화 실행

## 13. 상태 머신

```
view-mode → (edit click) → edit-mode (dirty = false)
edit-mode → (any field change) → edit-mode (dirty = true)
edit-mode + dirty → (save) → saving → view-mode (dirty = false)
                           → (cancel) → confirm dialog → view-mode (discard changes)
```

## 14. API

- `GET /api/me/avatar` — 현재 아바타 상태
- `PATCH /api/me/avatar` — 부분 업데이트 (body: 변경된 필드만)
- `POST /api/me/avatar/reset` — 초기화
- `POST /api/me/avatar/relearn` — GPT 재학습 시작 (새 Connect Code 발급)

## 15. 엣지 케이스

- **스탯 조정 도중 매칭 진행 중**: 경고 배너 "진행 중인 세션이 있습니다. 스탯 변경은 다음 세션부터 적용됩니다."
- **태그 8개 초과 시도**: 새 태그 입력 차단, 툴팁 "최대 8개까지"
- **이니셜 수동 입력 중 특수문자**: 실시간 필터링, 2글자 초과 자동 자르기
- **백그라운드 동기화 충돌** (다른 탭에서 변경): 저장 시 서버 응답의 `updatedAt` 체크, mismatch 시 "다른 곳에서 변경되었습니다. 최신 상태를 불러올까요?"

## Design Rationale

### 결정: 편집 모드와 읽기 모드 **분리**

- **이유**: 실수로 슬라이더를 건드려 스탯이 바뀌는 사고 방지. 편집은 명시적 의도가 필요.
- **대안**: 항상 편집 가능 → 기각 (스탯은 매칭 알고리즘에 직접 영향, 신중해야 함)

### 결정: 초기화 시 **닉네임 타이핑 확인**

- **이유**: GitHub/Linear의 repo/org 삭제 UX 패턴. 되돌릴 수 없는 작업은 "의식적인 저항"을 요구.
- **대안**: 체크박스 확인 → 기각 (체크박스는 무의식적으로 클릭 가능)
